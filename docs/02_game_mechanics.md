# PROJECT: OVERRIDE - 게임 시스템 설명

## 1. 게임의 3가지 핵심 사이클

### The Input: "사건의 재구성"

**출처:** 실제 변리사 1차/2차 기출문제

**게임플레이:**

```
기출문제의 사실관계
    ↓
게임 내 "의뢰인 NPC"의 사연으로 변환
    ↓
"변리사님! 제가 10년 전에 산 땅이 알고 보니 남의 땅이라는데요?"
(취득시효 관련 민법 문제)
```

**상황 예시:**

```
[2024 변리사 1차 실제 기출문제]

문제: A가 20년 동안 B의 토지를 공연하게, 평온하게, 선의로 점유했다.
      A는 취득시효로 토지 소유권을 얻을 수 있는가?

↓ (게임화)

[NPC: 농부 김철수]
"변리사님! 제가 20년 동안 이 땅을 농사지어왔는데,
 갑자기 원래 주인이 나타나서 땅을 내놔라고 합니다.
 제가 이 땅의 소유자가 될 수 있을까요?"

→ [플레이어 응답 시스템]
```

## 2. The Process: "청구항 크래프팅 & 법률 배틀"

### A. 청구항 크래프팅 (Claim Crafting)

**메커니즘:** 특허 명세서를 작성할 때, 구성요소 블록을 조립하는 방식

**블록 구조:**

```
┌─────────────────────────────────┐
│   특허 청구항 크래프팅 인터페이스 │
├─────────────────────────────────┤
│                                 │
│  [구성요소 A]  [구성요소 B]      │
│      ↓              ↓           │
│   [연결 관계] → [기능]          │
│      ↓                          │
│   [모든 필수 요소 충족?]         │
│   ☐ 신규성 요소 확인            │
│   ☐ 진보성 요소 확인            │
│   ☐ 명세서 기재 요소 확인        │
│                                 │
│   [제출] [재검토]               │
└─────────────────────────────────┘
```

**All Elements Rule:**

```python
def validate_claim(claim_elements):
    required_elements = {
        '구성요소': {},
        '연결관계': {},
        '기능설명': {},
        '신규성근거': {},
        '진보성근거': {}
    }

    for required in required_elements:
        if required not in claim_elements:
            raise ClaimError(f"필수 요소 누락: {required}")
            # → 화면에 빨간 경고 이펙트 표시

    return claim_elements  # ✓ 성공
```

블록 하나라도 빠지면 **시스템 경고(Error)** 발생 → 플레이어는 수정해야 함.

### B. 법률 배틀 (Legal Battle)

**배틀 구조:**

```
┌───────────────────────────────────────────┐
│          법률 배틀 인터페이스              │
├───────────────────────────────────────────┤
│                                           │
│  [심사관 AI]                [플레이어]    │
│  ┌──────────┐            ┌──────────┐    │
│  │ 판례 검색 │            │ 예외근거 │    │
│  │   중...   │ ←battle→  │   검색   │    │
│  │ RAG 진행중│            │   중...   │    │
│  └──────────┘            └──────────┘    │
│                                           │
│  AI 공격 메시지:                          │
│  "대법원 2020다123456 판결에 따르면      │
│   너는 패소야!"                           │
│  ⚠️ [이의 제기하기] [항소하기]          │
│                                           │
│  플레이어 반격:                           │
│  "원칙적으로는 맞지만, 이 경우는         │
│   '선의의 제3자'에 해당하므로 예외다!"   │
│  ✓ [Key-phrase Match] 승리!             │
│                                           │
└───────────────────────────────────────────┘
```

**배틀의 3단계:**

1. **AI 심사관 공격 (Enemy Turn)**
   - RAG: 판례 데이터베이스에서 관련 판결 검색
   - LLM: "이 사건은 판례 OOO와 동일하므로 패소"
   - 플레이어에게 법적 문제 제시

2. **플레이어 반격 (Player Turn)**
   - 카드 선택: "선의의 제3자", "비용 부담", "특수 상황" 등의 예외 카드
   - Key-phrase 검색: 올바른 법적 근거 입력
   - 입력된 텍스트가 AI의 기준과 매칭되면 승리

3. **결과 처리 (Resolution)**
   - **승리:** "Override 성공! 이 예외사유가 인정되었습니다."
   - **패배:** "이것은 예외사유로 인정되지 않습니다. 다시 시도하세요."
   - **점수 업데이트:** 플레이어의 "법률 실력" 레벨 증가

**배틀 예시:**

```
[기출문제]
A가 B의 토지를 20년 점유했을 때, 취득시효가 성립하는가?

[게임 진행]

1단계: 플레이어가 "취득시효 성립"이라고 주장
   → AI 심사관: "잠깐, 너는 선의인가?"

2단계: 플레이어가 "선의인지 확인할 근거가 없다"고 응답
   → AI 심사관: 판례 검색...
   → "대법원 2018다250000: 취득시효 성립 전제 = 선의 + 평온성"
   → AI: "넌 선의 요건을 입증하지 못했으므로 패소!"

3단계: 플레이어가 "하지만 이 사건은 동일한 점유자이므로,
             민법 제197조 '등기 없는 부동산 취득'의 특례가 적용됩니다"
   → Key-phrase Match: [취득시효] + [선의] + [평온성] ✓
   → 게임: "정답! Override 성공! 취득시효 성립 인정!"
   → 경험치 +10, 법률 이해도 +5%
```

## 3. The Visuals: "Juicy Feedback"

### A. 타격감 (Impact Feedback)

**목표:** 정답을 맞혔을 때의 쾌감을 극대화

**이펙트:**

```
정답 키워드(Key-phrase) 입력
    ↓
화면이 깨지는 이펙트 (셔터 효과)
    ↓
텍스트가 폭발하는 이펙트
    ↓
"OVERRIDE SUCCESS!" 메시지
    ↓
경험치 + 사운드 + 진동 피드백
```

**구현 예시:**

```python
class FeedbackSystem:
    def on_correct_answer(self, keyword):
        # 시각적 효과
        self.screen.shake(intensity=5, duration=0.5)
        self.screen.break_effect(position=center)
        self.text.explode_text(keyword)

        # 청각적 효과
        self.audio.play_sound("victory_chime.wav")

        # 햅틱 피드백 (모바일)
        self.haptic.vibrate(pattern=[200, 100, 200])

        # 게임 시스템
        self.player.gain_xp(10)
        self.battle.end_battle(winner="player")
```

## 4. 게임 진행 흐름 (Game Flow)

```
1. [NPC 등장]
   의뢰인이 법적 문제를 가지고 찾아옴

2. [사실관계 정리]
   게임: 사건의 주요 사실을 시각화
   학습: 판례의 "사실관계" 섹션 읽음

3. [법적 주장 수립]
   게임: 청구항 크래프팅 (블록 조립)
   학습: 관련 법 조문 검색 및 정리

4. [심사관과의 배틀]
   게임: Turn-based 법률 배틀
   학습: AI가 제시한 반박에 대한 대응 논리 수립

5. [판정 및 보상]
   게임: 승리/패배 판정
   학습: 자신의 응답이 왜 맞았는지 분석

6. [경험 누적]
   게임: XP 획득, 레벨 상승
   학습: 같은 유형의 문제가 나올 때 빠른 판단 가능
```

## 6. 게임 메커니크의 법률학적 의미

### 왜 이 게임 메커닉이 효과적인가?

| 게임 요소 | 학습 심리학 | 법률 이해 |
|----------|-----------|---------|
| **NPC 스토리** | 맥락 기억 향상 | 추상적 판례를 구체적 사건으로 이해 |
| **블록 조립** | 구조적 사고 | 법 요건의 '조합'이 핵심임을 이해 |
| **AI 배틀** | 즉각적 피드백 | 자신의 법적 주장이 타당한지 검증 |
| **Turn 기반 시스템** | 속도 조절 | 각 법적 개념을 깊이 있게 사고 |

## 7. 핵심 게임플레이 루프

```
┌─────────────────────────────────────────────┐
│                                             │
│  Read Problem (기출문제 읽기)                │
│         ↓                                   │
│  Craft Claim (법적 주장 수립)                │
│         ↓                                   │
│  Battle with AI (심사관과 배틀)              │
│         ↓                                   │
│  Get Feedback (즉각적 피드백 수신)           │
│         ↓                                   │
│  Refine Logic (논리 재검토)                  │
│         ↓                                   │
│  [Learning Complete] + XP Gain             │
│         ↓                                   │
│  Next Problem (다음 문제로)                  │
│                                             │
└─────────────────────────────────────────────┘
```

이것이 TDD 학습법의 게임화된 형태입니다.
