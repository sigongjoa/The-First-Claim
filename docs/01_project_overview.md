# PROJECT: OVERRIDE - 프로젝트 전체 개요

## 1. 프로젝트 정체성 (Identity)

### 프로젝트명: OVERRIDE

**의미:** 기존의 판례/심사 결과를 뒤집고, 나의 논리로 재정의한다

### 장르: Legal Logic Defense Simulation / Serious Game

### 핵심 철학

> "법을 읽는(Read) 사용자가 아니라, 법을 설계하는(Architect) 창조자가 된다."

이는 단순히 법 조문을 암기하는 것이 아니라, 법 체계의 논리 구조를 이해하고 새로운 상황에 적용할 수 있는 능력을 기르는 것을 의미합니다.

---

## 2. 이중 목표 (Dual Goals)

### Goal 1: 시험 합격 (Exam Success)

**대상:** 변리사 1차, 2차 시험

**과목:**
- 민법 (제1조 ~ 제184조 민법총칙 중심)
- 특허법 (신규성, 진보성, 명세서 해석 중심)

**성공 기준:**
- 변리사 시험 고득점 합격
- 기출문제 90% 이상 정답률

### Goal 2: 커리어 성장 (Career Development)

**포지셔닝:** AI와 Legal Tech를 완벽하게 이해하는 '대체 불가능한 변리사'

**차별화:**
- 법이론만 아는 변리사 vs. **AI로 법을 설계할 수 있는 변리사**
- 포트폴리오: 이 프로젝트 자체가 AI Legal Tech 포트폴리오

---

## 3. 프로젝트의 혁신성

### 기존 수험 방식의 문제점

```
[이론 강의 듣기] → [기본서 읽기] → [문제 풀이]
↓
수동적, 지루함, 즉시 잊음
```

### PROJECT: OVERRIDE의 혁신

```
[기출문제를 먼저 본다] → [문제 해결에 필요한 지식만 습득] → [게임 로직으로 구현]
↓
능동적, 재미있음, 장기 기억
```

**핵심:** TDD (Test-Driven Development) 학습법

---

## 4. 게임으로서의 가치

### 학습 심리학 활용

| 요소 | 효과 |
|------|------|
| **게임화** | 반복 학습을 '게임 플레이'로 인식 → 보상 회로 활성화 |
| **즉각적 피드백** | 정답/오답 결과를 화려한 이펙트로 표현 → 신경 가소성 향상 |
| **나쁜으로의 몰입** | 법 문제를 'NPC의 의뢰'로 표현 → 맥락 이해도 향상 |
| **점진적 난이도** | Phase 1(민법) → Phase 4(실전 시험) → 단계별 성장 |

### 역전재판 + 하스스톤 + 변호사

```
UI 컨셉: 역전재판의 법정 느낌
      + 하스스톤의 카드 덱 빌딩
      = 변리사 수험용 법률 방어 게임
```

---

## 5. 개발자다운 접근법

### 왜 변리사 시험에 '코딩'이 필요한가?

1. **사고 구조화:** 법 조문을 변수, 조건, 로직으로 변환 → 뇌가 법을 더 깊게 이해
2. **자동화:** 판례 데이터를 RAG로 검색 → 시험장에서 '내 머릿속 AI 심사관' 작동
3. **포트폴리오:** 코딩 포트폴리오 + 법률 자격증 → 고용 시장에서 '대체 불가능'해짐

### 기술 스택

| 역할 | 기술 |
|------|------|
| **백엔드** | Python + LLM (GPT/Llama) |
| **지식베이스** | Vector DB (Pinecone/Weaviate) |
| **프론트엔드** | Unity 또는 Web (React) |
| **데이터** | 국가법령정보센터 + 기출문제 크롤링 |

---

## 6. 프로젝트의 4단계 구조

### Phase 1: 민법의 기초 (Logic Gates)
- 개발: 텍스트 배틀 (Console)
- 학습: 민법총칙 완벽 이해

### Phase 2: 특허법의 구조 (System Architecture)
- 개발: 청구항 작성 시스템 프로토타입
- 학습: 특허 요건 및 명세서 작성법

### Phase 3: 판례의 시각화 (Database)
- 개발: RAG 시스템 + 화려한 UI
- 학습: 주요 판례 100개 암기 + 실전 감각

### Phase 4: Final Override (Deployment)
- 개발 중단 → 나 자신을 시험장에 배포
- 목표: 시험 합격 + AI Legal Tech 포트폴리오 완성
