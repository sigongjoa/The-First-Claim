## 🎯 핵심 개념 3가지

### 1️⃣ TDD 학습 사이클

```
Test (기출문제를 먼저 본다)
  ↓
Code (필요한 지식만 습득한다)
  ↓
Refactor (지식을 체계화한다)
  ↓
Pass (다시 풀어서 정답을 맞힌다)
  ↓
[도파민 분비! 다음 문제를 풀고 싶다]
```

**효과:** 90%+ 정답률 도달 가능

### 2️⃣ 4단계 게임 진행

```
Phase 1: 민법의 기초 (3-4주)
  → 텍스트 배틀, If-Then 로직

Phase 2: 특허법의 구조 (4-5주)
  → 청구항 크래프팅, 블록 시스템

Phase 3: 판례의 시각화 (5-6주)
  → AI 심사관, RAG 시스템, UI 이펙트

Phase 4: Final Override (2주)
  → 종합 시뮬레이션, 시험장 배포
```

### 3️⃣ 게임의 3가지 요소

```
The Input   → 사건의 재구성 (NPC의 의뢰)
The Process → 청구항 크래프팅 & 법률 배틀
The Visuals → Juicy Feedback (화면 이펙트)
```

## ❓ 자주 묻는 질문

### Q: 정말 12주에 가능할까요?
**A:** 하루 2-3시간 집중력 있게 학습하면 가능합니다. 단, 변리사 시험 기본 지식은 있어야 합니다.

### Q: 개발 경험이 없어도 되나요?
**A:** Python 기본 지식과 변리사 시험 기초는 필요합니다. 게임 개발 경험은 불필요합니다.

### Q: 게임은 정말 필요한가요?
**A:** 네. 게임화는 학습 심리학 기반으로, 정답률과 기억력을 크게 향상시킵니다.

### Q: AI 기능이 정말 도움이 될까요?
**A:** 네. AI는 판례 검색과 즉각적 피드백으로 학습을 3배 이상 가속화합니다.

### Q: 시험에 떨어지면 어떻게 하나요?
**A:** Phase별로 유연하게 조정할 수 있습니다. 기본은 건드리지 않고 심화 부분만 재구성하세요.

## 🎓 학습 일정 예시 (1주)

### Monday (월요일)
- 기출문제 3개 풀어보기 (오답)
- 틀린 이유 분석하기

### Tuesday (화요일)
- 관련 법 조문 검색
- 판례 3개 읽기
- 지식을 JSON으로 정리

### Wednesday (수요일)
- 파이썬 코드로 구현
- 테스트 코드 작성

### Thursday (목요일)
- 코드 리팩토링
- 같은 3개 문제 다시 풀기 (정답 확인)

### Friday (금요일)
- 같은 유형 다른 문제 5개 풀기
- 진도 확인 및 기록

### Saturday-Sunday (주말)
- 휴식 또는 추가 문제 풀기
- 주간 복습

### 성공의 핵심 3가지

1. **꾸준함**: 하루 2-3시간, 주 5일 이상
